안내서에 대하여

안내서의 대상

본 안내서는 JEUS®(이하 JEUS) MQ(Message Queue) 클라이언트 개발자와 JEUS MQ 서버 관리자를 대상으로 한다.

안내서의 전제 조건

Java Message Service에 대한 자세한 내용은 "Java Message Service Specification - version 2.0"과 관련 안내서를 참고한다.

JEUS의 기본적인 사용법과 제품을 이해하기 위해서는 다음의 안내서를 미리 숙지할 것을 권장한다.

  • JEUS 소개

  • JEUS 설치 및 시작하기

본 안내서의 모든 예제와 환경 구성은 UNIX의 스타일에 준하여 작성되어 Microsoft Windows™(이하 Windows)와 같이 다른 환경에서 작업하는 경우 몇 가지 사항을 고려해야 한다.

예를 들어 경로 구분자의 경우 UNIX 스타일인 “/”를 Windows 스타일인 “\”로 바꿔서 사용한다. 또한 환경변수도 Windows 스타일(%%)로 변경해서 사용한다. 본 안내서에 명시된 JEUS_HOME은 JEUS가 설치된 디렉터리이다.

안내서의 제한 조건

본 안내서의 내용은 Java 표준에 준해서 작성되었으나 제품 특성상 언급되는 Java EE나 Java 스펙에 대해서 자세히 다루지 않는다. 따라서 해당 내용은 Java 관련 문서를 참고한다. 실무에서의 구체적인 사용 방법이나 관리 및 운용에 관한 사항은 각 제품의 안내서를 참고한다.

안내서 구성

본 안내서는 총 6개의 장과 2개의 Appendix로 구성되어 있다.

  • “제1장 소개”

    Java Message Service에 대한 간단한 설명과 JEUS MQ의 특징에 대해서 설명한다.

  • “제2장 JEUS MQ 클라이언트 프로그래밍”

    JEUS MQ 클라이언트의 유형, 작성 절차 및 관리 객체에 대해서 설명한다.

  • “제3장 JEUS MQ 서버 설정”

    JEUS MQ 서버를 실행하기 위한 JMS 엔진과 리소스 설정 방법에 대해서 설명한다.

  • “제4장 JEUS MQ 클러스터링”

    JEUS MQ 서버의 부하를 줄이고 원활한 서비스를 제공하기 위해서 여러 대의 서버를 하나로 묶어서 사용하는 클러스터링에 대해서 설명한다.

  • “제5장 JEUS MQ 장애 극복”

    JEUS MQ에서 서버나 네트워크에 장애가 발생했을 때 복구하는 방법에 대해서 설명한다.

  • “제6장 JEUS MQ 특수 기능”

    JEUS MQ의 특수 기능인 Message Bridge, Message Sort, Global Order, Message Group, Message Management, Topic Multicast 기능에 대해 설명한다.

  • “Appendix A. Journal Store 추가 속성”

    Journal Store에 추가적으로 설정할 수 있는 속성들에 대해서 설명한다.

  • “Appendix B. JDBC Persistence Store 컬럼”

    JEUS MQ에서 Persistence store를 JDBC로 설정했을때 생성되는 테이블의 컬럼에 대해 설명한다.
