jeus.io
Class Selector 

java.lang.Object
  jeus.io.Selector
public class Selector
extends java.lang.Object

Selector는 IOComponentCreator를 통해 생성되며, 그 IOComponentCreator가 생성될때 지정된 타입에 맞는 Selector가 생성된다. SelectorType의 종류는 다음과 같다.

  • IOComponentCreator.BLOCKING_SOCKET
  • IOComponentCreator.NON_BLOCKING
  • IOComponentCreator.LOCAL

LOCAL은 local에 있는 대상과 통신할때 최적화를 위한 설정이며, 나머지는 blocking방식을 사용할지, non-blocking방식을 사용할지, 또는, socket을 사용할지 channel을 사용할지를 선택하는 것이다. IOComponentCreator를 생성할 때 에는 어플리케이션이 실행되는 환경에 따라 알맞는 SelectorType을 선택해야 한다.

예를 들어 JDK1.3 환경에서는 BLOCKING_SOCKET만 선택 가능하다. JDK1.4이상 환경에서는 2가지 SelectorType이 모두 사용가능 하나 가능하면 CHANNEL을 사용하는 SelectorType을 사용하는 것이 성능에 더 좋다.

Selector는 본래 non-blocking IO를 지원하기 위한 multiplexor를 지칭하는 용어이나, JEUS NETWORK LIBRARY는 blocking io와 non-blocking io를 모두 지원하는 통합 라이브러리이므로 여기서 말하는 Selector 의 의미는 본래 의미보다 약간 확장되었다. non-blocking io의 경우는 본래 multiplexor의 의미대로 사용되지만 blocking io의 경우에는 좀 다르게 사용된다.

blocking io에서는 상대방의 소켓으로 부터 언제 데이터가 보내지는지를 감지할 수 없으므로 입력 스트 림이나 채널의 read 메소드를 호출하여 데이터가 올 때 까지 블록킹이 되어야만 한다. 때문에 보통 별도 의 읽기 전용 쓰레드를 만들어서 사용하고 blocking io에서의 Selector는 이런 읽기 쓰레드들을 생성하고 실행시키는 일을 주로 하게된다. blocking selector에서는 읽기 쓰레드들을 생성하기 위한 쓰레드풀 설정 이 별도로 필요하다. 이는 IOComponentCreator를 생성할때 설정할 수 있으며, 별다른 설정이 없을시에는 System thread pool이 사용된다.
