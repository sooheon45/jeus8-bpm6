createSelector

public Selector createSelector(java.lang.String name,
                               int selectNum,
                               boolean useDualSelector)
                             throws java.io.IOException
IOComponentCreator의 Selector를 생성하는 메소드이다. 첫번째 인수는 Selector 내부적으로 사용될 쓰레드의 이름을 지정하는 것이고 selectNum은 NIO_SELECTOR를 사용할 때 Selector를 여러개 설정할 경우 사용한다. 세번째 인수인 useDualSelector는 read, write이벤트를 별도의 selector에서 할지 여부를 선택하는 인수이다. Selector는 Acceptor로 부터 생성된 StreamHandler를 Selector의 이벤트 핸들러로 등록하면 Selector에서 발생한 IOEvent들에 대응하는 StreamHandler의 메소드들을 호출하는 역할을 한다.

Parameters:
name - selector의 이름. 이 이름에 "-Selector"를 붙인것이 selector thread의 이름이 된다.
selectNum - selector의 갯수.
useDualSelector - read와 write를 별도의 java.nio.channels.Selector에서 할지를 지정한다.
Returns:
생성된 Selector 
Throws:
java.io.IOException - selector 생성 실패

createAcceptor

public Acceptor createAcceptor(java.lang.String listenHost,
                               int port,
                               jeus.io.SocketListener listener,
                               javax.net.ssl.SSLContext sslContext,
                               jeus.util.protocol.ssl.SSLConfig sslconf,
                               int backlog)
                             throws java.io.IOException
IOComponentCreator의 Acceptor를 생성하는 메소드이다. 첫번째 인수는 ServerSocket을 바인드할 포트이고 두번째 인수 SocketListener는 Acceptor가 Socket연결을 받아 들인 후 이 이벤트를 전달할 이벤트 리스너이다. 이전에 Endpoint를 사용할 때는 이 이벤트는 내부적으로 처리되고 SocketStream이라는 것이 만들어진 후 ConnectionListener로 생성하여 connectionEstablished메소드를 호출하였었다.

Parameters:
listenHost - listen받을 host
port - listen받을 port
listener - Socket이 accept되는 이벤트를 수신하는 리스너
sslContext - acceptor가 사용할 javax.net.ssl.SSLContext 
sslconf - Acceptor가 사용할 SSL의 설정(jeus.util.protocol.ssl.SSLConfig 참조)
backlog - 바인드할 ServerSocket에서 사용할 backlog length
Returns:
생성된 Acceptor
Throws:
java.io.IOException
