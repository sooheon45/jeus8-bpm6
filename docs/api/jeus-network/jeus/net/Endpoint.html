jeus.net
Class Endpoint 

java.lang.Object
  jeus.net.Endpoint
public class Endpoint
extends java.lang.Object
Endpoint는 네트워크 통신에 필요한 양쪽 종단, Connector와 Acceptor를 표현하는 객체이다. Acceptor로 동작할 경우에는 Endpoint를 생성할 때 서버 소켓에 바인드할 주소를 설정하고 Endpoint.export()를 호출하여 서버를 실행시키며, Connector로 동작할 경우에는 Endpoint.trySocketStream(SocketID,Object)를 호출하여 서버와 연결을 한다. 클라이언트에서 연결을 시도하면 서버는 AcceptorConnectionListener를 생성하여 클라이언트 요청들을 처리하고 클라이언트는 ConnectionListener를 생성하여 서버의 요청/응답들을 처리한다. 상대방과의 데이터를 송수신은 SocketStream에서 이뤄진다. 마지막으로 연결을 종료할 때는 Endpoint.unexport()를 호출하였다. Server들 사이에 서로 연결을 맺고 peer-to-peer 방식으로 통신하기 위해 사용한다. Application은 이 class를 통해 physical connection을 의미하는 socket stream을 얻는다.

See Also:
Acceptor, Connector, jeus.net.connection.EndpointConnector

broadcast

public boolean broadcast(int connectionType,
                         SocketStream sockStream,
                         java.lang.Object message)
연결되어 있는 SocketSteam을 통해 모두에게 메시지를 broadcast한다. 부번째 인수의 SocketStream을 제외한 현재 이 Endpoint에 접속한 다른 SocketStream으로 메시지를 전송할 때 사용되는 메소드이다. 그런데 필요에 따라 클라이언트들을 그룹을 지어 관리하고 각각의 그룹별로 broadcast할 경우가 있을 것이다. 이런경우 각각의 그룹을 ConnectionType이라는 속성으로 구분할 수 있다. 이 값은 어플리케이션 개발자가 정하는 것이며 서버와 클라이언트는 이 값을 서로 약속하고 사용해야한다. 지금까지의 예제에서 Endpoint가 Connector로 사용될 경우 SocketID를 지정하지 않았는데 이것은 SocketID가 지정되지 않은 것이 아니라 내부적으로 NoListenID라는 객체를 설정한 것이고 이 SocketID는 ConnectionType의 값으로 SocketID.CONNECTION_CLIENT_TYPE를 사용했던 것이다. 만약 클라이언트들을 여러 그룹으로 나누어서 관리 하고 싶을 경우 NoListenID에 ConnectionType값을 지정하여 생성한 후 SocketID가 인수로 있는 Endpoint의 생성자나 setSocketID라는 메소드를 호출해 주어야 한다.
