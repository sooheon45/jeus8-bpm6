jeus.workarea
Interface UserWorkArea 


      
public interface UserWorkArea
사용자 Work Area에 접근하기 위한 인터페이스.
      

      실제 UserWorkArea 객체는 JNDI 이름 "java:comp/UserWorkArea"를 lookup해서 가져올 수 있다.
      

      Work Area 서비스는 프로그램이 어떤 묵시적(implicit)인 컨텍스트를 계속 전달하고자 할 때 사용할 수 있는
      기능이다. 마치, security 컨텍스트나 트랜잭션 컨텍스트와 같이 별도의 인자로 전달하지 않아도 로컬 
      혹은 원격 메소드 호출시에 계속 전달된다. 
      사용자 컨텍스트를 전달해야 하는 경우나 메소드의 인자수가 많아지는 경우에 Work Area를 사용할 수 있다.
      
 
      Work Area는 일종의 사용자 저장 공간으로 name-value pair 형태의 맵(Map)으로 저장된다. Work
      Area를 새로 시작하게 되면 현재 쓰레드에 같이 따라 다니기 때문에 호출된 메소드나 EJB 와 같은 컴포넌트에서 
      계속 사용할 수가 있다. 또한, 원격 EJB 호출을 하는 경우에도 전달(propagation) 되는 특성이 있다.
      
 
      Work Area는 마치 스택처럼 중첩하여 열 수 있는데 이렇게 하면 기존 값을 지우지 않고 잠시 새로운 값을 넣거나
      하는 식으로 사용될 수 있다.

public void complete()
                   throws NoWorkAreaException,
