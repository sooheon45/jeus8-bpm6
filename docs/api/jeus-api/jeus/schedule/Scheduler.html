jeus.schedule
Interface Scheduler 


         
All Superinterfaces:
java.rmi.Remote

         
      

      

      
public interface Scheduler
extends java.rmi.Remote
Scheduler는 작업을 등록하는 인터페이스를 제공한다.
      실제(concrete) Scheduler instance는 SchedulerFactory를 통해서나
      JNDI Lookup을 통해서 얻을 수 있다.

         

            

            
See Also:
ScheduleListener, Schedule, java.util.Timer

cancel
public void cancel()
                 throws java.rmi.RemoteException

            
등록된 모든 Schedule 작업을 종료시킨다(graceful termination).
               이는 호출한 시점까지 등록되어 있던 작업을 종료시킨다.
               이 method 호출 이후에도 새로운 작업을 등록할 수 있다.

registerSchedule
public ScheduleController registerSchedule(java.lang.String taskClassName,
                                           java.util.Date firstTime,
                                           long period,
                                           java.util.Date endTime,
                                           long maxcount,
                                           boolean isDaemon)
                                         throws java.lang.IllegalArgumentException,
                                                JeusSchedulerException,
                                                java.rmi.RemoteException

            
특정 시간에 시작되어 fixed-delay로 반복 수행되는 작업을 등록한다.
               수행 종료 시간과 최대 수행 횟수를 지정할 수 있다.
               만약 주어진 시작 시간이 현재보다 과거이면 바로 수행되도록 한다.
               주어진 작업 클래스 이름이 올바른 ScheduleListener 객체가 아닌 경우 JeusSchedulerException이 발생한다.
            

            

            

               

                  
Parameters:
taskClassName - 등록할 작업으로 ScheduleListener 객체이어야 한다.
firstTime - 처음 수행될 시간.
period - 반복 수행 주기(milliseconds)
endTime - 종료 시간으로 이 시간 이후에는 작업이 더이상 수행되지 않는다. null인 경우에는 지정하지 않는다.
maxcount - 최대 수행 횟수. UNLIMITED_COUNT 인 경우에는 지정하지 않는다.
isDaemon - 원격 클라이언트의 경우에만 사용되며 true이면 연결이 끊어지면 작업을 자동종료 한다.
                     

                  
                  
Returns:
등록된 작업을 제어하기 위한 Handle.

                  
                  
Throws:
java.lang.IllegalArgumentException - if task or firstTime is null, or period is non-positive
JeusSchedulerException - 내부적으로 작업 등록이 실패한 경우에 발생
java.rmi.RemoteException - 원격 클라이언트에서 호출시 통신상의 오류가 난 경우에 발생
                     

                  
                  
Since:
5.0

registerSchedule
public ScheduleController registerSchedule(java.lang.String taskClassName,
                                           java.util.Hashtable parameters,
                                           boolean isDaemon)
                                         throws JeusSchedulerException,
                                                java.rmi.RemoteException

            
작업을 등록한다.
               등록된 작업의 첫 수행시간을 결정하기 위해 Schedule.nextTime()이 먼저 호출되며,
               수행시간이 되면 Schedule.onTime()이 호출되고 수행이 끝나면 Schedule.nextTime()을
               호출하여 다음 수행시간을 결정한다.
               만약 Schdule.nextTime()이 현재보다 과거 시간을 리턴하면 작업은 바로 수행되도록 한다.
               주어진 작업 클래스 이름이 올바른 Schedule 객체가 아닌 경우 JeusSchedulerException이 발생한다.
            

            

            

               

                  
Parameters:
taskClassName - Schedule 객체의 클래스 이름.
parameters - RemoteSchedule을 초기화 할 때 넘겨가는 값의 집합.
isDaemon - 원격 클라이언트의 경우에만 사용되며 true이면 연결이 끊어지면 작업을 자동종료 한다.
                     

                  
                  
Returns:
등록된 작업을 제어하기 위한 Handle.

                  
                  
Throws:
JeusSchedulerException - 내부적으로 작업 등록이 실패한 경우에 발생
java.rmi.RemoteException - 원격 클라이언트에서 호출시 통신상의 오류가 난 경우에 발생

주어진 시간(time)과 주기를 계산하여 현재 이후 첫번째 주기시간을 리턴한다.
               이는 작업의 첫번째(first time) 스케쥴링 시간을 조정하기 위해 사용자가 사용하는 유틸리티 method이다.
               주어진 시간이 과거인 경우에 의미가 있으며,
               주어진 시간이 현재 이후이면 동일한 시간이 리턴된다.
            

            

            

               

                  
Parameters:
time - 시작시간
period - 주기
                     

                  
                  
Returns:
현재 이후 첫번째 주기시간. 항상 새로운 Date object가 리턴된다.

                  
                  
Throws:
java.lang.IllegalArgumentException - if time is null or period is non-positive
java.rmi.RemoteException - 원격 클라이언트에서 호출시 통신상의 오류가 난 경우에 발생
