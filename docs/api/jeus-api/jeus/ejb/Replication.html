jeus.ejb
Annotation Type Replication 


         

      

      

      
@Target(value=METHOD})
@Retention(value=RUNTIME)
public @interface Replication
클러스터링 환경에서 해당 EJB 빈의 상태를 어떻게 방식으로 복제(replication)할 지를 설정한다.
      

      현재 Synchronous, Asynchronous 방식을 지원하며 기본적으로는 Synchronous 방식이 사용되며, 
      idempotent 메소드는 복제가 이루어지지 않는다.
      
    
      
  • Synchronous(SYNC) - 메소드가 호출이 끝난 시점에 백업 세션 서버로 복제가 이루어진다.
    • 
      
  • Asynchronous(ASYNC) - 메소드 호출이 끝나고 리턴이 된 이후에 비동기적으로 복제가 이루어진다.
    • 
      
  • None - 해당 메소드 호출에 대해서는 복제를 하지 않는다.
    • 
      

      Synchronous 방식은 메소드 호출이 끝나면 바로 복제가 되기 때문에 좀더 세션 상태를 최신으로 유지하는데 도움이 되지만,
      Asynchronous 방식에 비해 성능상의 단점이 있으므로 사용 환경에 맞게 적절히 선택하여 사용한다. 
      

      Stateful session 빈 클래스 혹은 빈 클래스에 있는 메소드에 대해서 이 어노테이션을 지정할 수 있으며,
      빈 클래스에 사용된 값은 명시적으로 설정을 하지 않은 메소드에 대해서 적용된다.
