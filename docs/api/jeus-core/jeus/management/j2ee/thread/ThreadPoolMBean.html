jeus.management.j2ee.thread
MBean ThreadPoolMBean

ObjectName Pattern:
JEUS:j2eeType=JeusService,jeusType=ThreadPool,*
Permission Name:
{parent-permission-name}.threadpool.{threadpool-name}
All Superinterfaces:
jeus.management.j2ee.J2EEManagedObjectMBean, jeus.management.j2ee.StatisticsProvider
MBean ThreadPoolMBean
JEUS에서 사용하는 쓰레드 풀을 관리한다.

기본적으로 JDK에 존재하는 java.util.concurrent.ThreadPoolExecutor의 의미를 따른다.

Operation Detail

allowCoreThreadTimeOut

 void allowCoreThreadTimeOut(boolean value)
core 쓰레드들에 타임아웃을 적용하여 쓰레드의 상태가 idle일 때, keep-alive time에 따라 쓰레드를 종료, 정리할지의 정책 여부를 설정한다.

true로 설정할 경우 non-core 쓰레드들에 적용되는 keep-alive정책이 그대로 적용이 되어, core pool size 아래로까지 쓰레드들이 줄어 들 수 있다.

false로 설정할 경우 core 쓰레드들은 결코 종료되지 않고, 비록 idle상태일지라도 core 쓰레드들은 계속 유지된다.

Parameters:
value - true core 쓰레드들에 타임아웃을 적용하고자 할 경우, 그렇지 않다면 false
Since:
JEUS 6 Fix#6

allowsCoreThreadTimeOut

 boolean allowsCoreThreadTimeOut()
keep-alive 타임아웃이 발생하면서 쓰레드가 idle상태일 때, core pool size 아래로까지 쓰레드들을 종료, 정리할지의 여부를 나타낸다.

리턴 값이 true일 경우 non-core 쓰레드들에 적용되는 keep-alive 정책이 그대로 적용이 되어, core pool size 아래로까지 쓰레드들이 줄어 들 수 있다.

리턴 값이 false일 경우 core 쓰레드들은 결코 종료되지 않고, 비록 idle상태일지라도 core 쓰레드들은 계속 유지된다.

Returns:
true core 쓰레드들에 다임아웃을 적용할 경우, 그렇지 않다면 false
Since:
JEUS 6 Fix #6

interruptWorker

 void interruptWorker(long tid, boolean forcefully)
쓰레드 아이디(tid)에 해당하는 해당 쓰레드에 인터럽트(Interrupt)를 일으킨다.

인터럽트 대상은 태스크를 수행하고 있는 쓰레드일 수도 있고, 조건에 따라 idle 상태의 쓰레드만 인터럽트를 허용할 수 있다. 보통, 문제가 있는 블럭된(blocked) 쓰레드에 대해 사용자가 명시적으로 해당 쓰레드를 강제 종료시키기 위해 사용될 수 있다.

Parameters:
tid - 인터럽트를 일으키려는 쓰레드의 tid
forcefully - true일 경우 해당 쓰레드가 비록 태스크를 수행하고 있더라도 즉시 그 쓰레드에 대해 인터럽트를 수행한다. false일 경우 오직 해당 쓰레드가 idle 상태일 경우만 인터럽트를 수행한다.
Since:
JEUS 6 Fix#6
